**Script 1: `scripts/train/01_generate_data.py`**

Generate 300-500 labeled synthetic GTD training examples per atom type (5 types: task, fact, event, decision, insight) via Anthropic Claude API. Use structured outputs (`output_config.format` with `json_schema`) — NOT legacy JSON mode.

Implementation requirements:
- Load `ANTHROPIC_API_KEY` from `.env.local` in the repo root using `python-dotenv` (the project already uses `.env.local` for secrets). Use `load_dotenv(dotenv_path=Path(__file__).resolve().parents[2] / '.env.local')` to find it relative to the script.
- Use `claude-haiku-4-5` model for cost efficiency (batch generation of 1500-2500 examples).
- Define a JSON schema for structured output:
  ```python
  SCHEMA = {
      "type": "object",
      "properties": {
          "text": {"type": "string", "description": "the GTD inbox item text"},
          "label": {"type": "string", "enum": ["task", "fact", "event", "decision", "insight"]},
      },
      "required": ["text", "label"],
      "additionalProperties": False
  }
  ```
- Use `output_config={"format": {"type": "json_schema", "schema": SCHEMA}}` in the `client.messages.create()` call.
- Define GTD type definitions for the system prompt so the LLM understands each category:
  - task: requires a concrete next action; has a clear completion state
  - fact: reference information; no action required; states something true
  - event: time-anchored; describes something that will or did happen
  - decision: records a choice already made; may include rationale
  - insight: generalizable principle or learning; abstracted from a specific situation
- Rotate through at least 5 style variants per label to ensure diversity (per RESEARCH.md pitfall 5):
  1. Short fragment (1-5 words, no punctuation)
  2. Complete professional sentence
  3. Abbreviated telegram/text message style
  4. With typo or informal capitalization
  5. Genuinely ambiguous (could fit multiple GTD types) — force the label but make the text borderline
- Generate TARGET_PER_LABEL examples per label (default 400, configurable via `--count` CLI arg).
- Write output to `scripts/training-data/type-classification.jsonl` (one JSON object per line: `{"text": "...", "label": "..."}`).
- Use `tqdm` for progress bar.
- Add `time.sleep(0.05)` between API calls to avoid rate limits.
- Validate: only write examples where the returned label matches the requested label (LLM may occasionally return wrong label despite schema constraint — log and skip these).
- At the end, print a summary: count per label, mean/median character length per label, count of short examples (<10 chars) per label.
- Support `--resume` flag: if JSONL file already exists, count existing examples per label and only generate the remaining needed.
- Handle API errors gracefully with retry (max 3 retries per example with exponential backoff).

**Script 2: `scripts/train/02_embed_data.py`**

Embed the generated JSONL training data using the same MiniLM model the browser uses.

Implementation requirements:
- Load `scripts/training-data/type-classification.jsonl` and parse all examples.
- Use `SentenceTransformer('sentence-transformers/all-MiniLM-L6-v2')` — this is the Python-side equivalent of the browser's `Xenova/all-MiniLM-L6-v2`.
- CRITICAL: Call `model.encode(texts, normalize_embeddings=True, batch_size=64, show_progress_bar=True)` — the `normalize_embeddings=True` flag is mandatory for parity with the browser worker's `{ pooling: 'mean', normalize: true }`.
- Save embeddings to `scripts/train/embeddings_cache.npy` (this file is gitignored — it's ~2.3MB and reproducible).
- Save labels to `scripts/train/labels_cache.npy` (integer-encoded via sklearn LabelEncoder, also gitignored).
- Save the label-to-index mapping to `scripts/train/label_map.json` (committed — needed by script 03 and the browser).
- Run parity verification after encoding:
  1. Assert output shape is `(N, 384)` with dtype `float32`
  2. Assert every vector has L2 norm within 1e-5 of 1.0 (normalized)
  3. Print: "Parity check passed: {N} embeddings, 384-dim, normalized float32"
- Print final summary: total examples embedded, examples per label, embedding matrix shape.
