   - Otherwise, render the existing single-type display unchanged

```tsx

          <AtomTypeIcon type={props.suggestion.suggestedType} size={14} />
```

**IMPORTANT:** The existing "Header row" (type icon + type name + section + AI badge), "Reasoning" section, and "Expand button" should ONLY show for the confident (non-ambiguous) case. - Ambiguous path shows: ambiguous buttons (with accept built-in) + related atoms
- Confident path shows: header + reasoning + expand + related atoms + accept/dismiss

For the ambiguous path, clicking either type button both selects the type AND accepts, so the separate Accept/Dismiss buttons are not needed. The Dismiss button should still be available (user may not want either suggestion).
