**isReadOnly enforcement (POLISH-05):**

In `src/ui/views/AtomDetailView.tsx`, add a read-only guard to prevent editing atoms with `isReadOnly: true`. Currently only `AnalysisAtom` has this flag, but the guard must work for any atom with the flag.

**Step 1: Add isReadOnly memo**

Near the top of the component body (after the `atom()` accessor is established), add:
```typescript
const isReadOnly = createMemo(() => {
  const a = atom();
  return a != null && (a as Record<string, unknown>)['isReadOnly'] === true;
});
```

Import `createMemo` from `solid-js` if not already imported. This uses the same safe cast pattern as `getAtomDate()` already does in the same file (Pitfall 2 from RESEARCH.md).

**Step 2: Guard all edit handlers**

Add `if (isReadOnly()) return;` as the first line in each of these handlers:
- `startEditTitle` (around line 113) — prevents entering title edit mode
- `handleStatusChange` (around line 145) — prevents status dropdown changes
- `handleDateField` (around line 155) — prevents date input changes
- `handleContentChange` (around line 186) — prevents content textarea changes
- Project `<select onChange>` handler (around line 401) — prevents project/section reassignment

Search for these function/handler names in the file and add the guard to each.

**Step 3: Disable input elements visually**

For all interactive elements (`<input>`, `<textarea>`, `<select>`) in the component, add `disabled={isReadOnly()}`. This provides the visual feedback that the atom cannot be edited.

For the title input/display area, add a conditional CSS class:
```tsx
class={`atom-detail-title-input${isReadOnly() ? ' atom-detail-readonly' : ''}`}
```

Add a simple CSS rule (inline in the component or in `layout.css`) for `.atom-detail-readonly`:
```css
.atom-detail-readonly {
  opacity: 0.7;
  cursor: not-allowed;
}
```

CRITICAL: Never destructure SolidJS props or state. Access atom data through `atom()` accessor, store data through `state.field`.
