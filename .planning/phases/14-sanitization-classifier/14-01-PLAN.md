  <action>
Create two Python scripts following the existing v3.0 training pipeline pattern.

**10_generate_sanitization_data.py** — Synthetic BIO-tagged NER data generator:
- Uses Faker library (not Claude API) to generate realistic PII entities for each of the 5 categories: PERSON, LOCATION, FINANCIAL, CONTACT, CREDENTIAL
- Template-based generation: define 40-60 atom-style text templates with entity placeholders (meeting notes, task descriptions, contact info, financial records, credential storage patterns)
- For each template, fill placeholders with Faker-generated data and compute character-level entity spans
- Convert character spans to word-level BIO tags (B-PERSON, I-PERSON, B-LOCATION, I-LOCATION, B-FINANCIAL, I-FINANCIAL, B-CONTACT, I-CONTACT, B-CREDENTIAL, I-CREDENTIAL, O)
- Also generate ~30% negative examples (text with no PII — generic GTD task descriptions, project notes)
- Target: 3000-5000 sentences with balanced entity category distribution
- Output format: JSONL to scripts/training-data/sanitization-ner.jsonl with structure: `{"tokens": ["Meeting", "with", "John", "Smith", ...], "ner_tags": [0, 0, 1, 2, ...]}`
- Tag mapping: O=0, B-PERSON=1, I-PERSON=2, B-LOCATION=3, I-LOCATION=4, B-FINANCIAL=5, I-FINANCIAL=6, B-CONTACT=7, I-CONTACT=8, B-CREDENTIAL=9, I-CREDENTIAL=10
- Save label map to scripts/train/sanitization_label_map.json
- CLI: `python 10_generate_sanitization_data.py --count 4000` (default 4000)
- Include variety: multi-word names, hyphenated locations, dollar amounts with commas, international phone formats, various API key patterns (sk-, ghp_, sk-ant-)

**11_train_sanitizer.py** — Fine-tune DistilBERT + ONNX export:
- Load data from sanitization-ner.jsonl using HuggingFace datasets
- Tokenize with `distilbert-base-cased` tokenizer
- CRITICAL: Use `tokenize_and_align_labels` pattern — only assign labels to first subword token of each word, set others to -100 (ignored in loss). This prevents BIO tag alignment crash with wordpiece tokens (Pitfall 2 from RESEARCH.md)
- Split 80/10/10 train/val/test
- Fine-tune with HuggingFace Trainer: AutoModelForTokenClassification with num_labels=11, 5 epochs, batch_size=16, evaluation_strategy="epoch", load_best_model_at_end=True, metric_for_best_model="overall_recall"
- Use seqeval for entity-level metrics (not token-level — Pitfall from RESEARCH.md)
- compute_metrics function using seqeval: precision, recall, f1 per entity category + overall
- RECALL GATE: After training, evaluate on test set. Print per-category recall. If overall recall < 0.85, print warning but continue (user decides)
- Export to ONNX via HuggingFace Optimum: `ORTModelForTokenClassification.from_pretrained()` then `save_pretrained()`
- Apply dynamic quantization (Q8) via onnxruntime quantization or optimum CLI
- Copy final model files to public/models/sanitization/: config.json, tokenizer.json, tokenizer_config.json, special_tokens_map.json, onnx/model_quantized.onnx (or model.onnx with quantization)
- Also save the label-to-id mapping as public/models/sanitization/sanitize-check-classes.json
- CLI: `python 11_train_sanitizer.py` (no arguments needed, reads from training-data/)
- Print final metrics summary: per-category precision/recall/F1, overall recall, model size in MB

Update scripts/train/requirements.txt to add: transformers, datasets, optimum[onnxruntime], seqeval, faker (keep existing deps).

Run the data generation script: `python scripts/train/10_generate_sanitization_data.py`
Then run the training script: `python scripts/train/11_train_sanitizer.py`

Note: Training requires a Python environment with GPU recommended but CPU works (slower). The existing venv at scripts/train/.venv can be extended.

**12_validate_sanitizer.mjs** — Browser-compatible ONNX NER validation:

- Configure for local model loading: `env.allowRemoteModels = false; env.allowLocalModels = true; env.localModelPath = './public/models/'`
- Create pipeline: `pipeline('token-classification', 'sanitization/sanitize-check', { dtype: 'q8' })`
- Use aggregation_strategy: 'simple' to get merged entity spans (not per-subword — Pitfall 6)

  1. "Meeting with John Smith at the office" — should detect PERSON "John Smith"
  2. "Call sarah.jones@gmail.com about the project" — should detect CONTACT

  4. "API key: sk-abc123def456ghi789" — should detect CREDENTIAL

- For each test: print input text, detected entities with category/confidence/span, and pass/fail
- Exit code 0 if all tests pass, 1 if any fail
- Print model load time and inference time per test case

CLI: `node scripts/train/12_validate_sanitizer.mjs`

Note: This script validates the model works in a JS runtime with Transformers.js — same pipeline that will run in the sanitization worker. It does NOT require a browser; Node.js with @huggingface/transformers works.
