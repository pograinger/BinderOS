**src/ai/sanitization/types.ts** — Core types and branded SanitizedPrompt:
- `declare const __sanitized: unique symbol;`

- `export type EntityCategory = 'PERSON' | 'LOCATION' | 'FINANCIAL' | 'CONTACT' | 'CREDENTIAL';`
- `export interface DetectedEntity { text: string; category: EntityCategory; start: number; end: number; source: 'ner' | 'regex' | 'both'; confidence: number; }`
- `export interface SanitizedResult { prompt: SanitizedPrompt; entities: DetectedEntity[]; entityMap: Map<string, string>; reverseMap: Map<string, string>; }` — entityMap maps pseudonym -> real text (for de-pseudonymization), reverseMap maps real text -> pseudonym (for quick lookup)
- `export function createSanitizedPrompt(text: string): SanitizedPrompt { return text as SanitizedPrompt; }` — ONLY function that casts to SanitizedPrompt
- `export interface EntityRegistryEntry { id: string; realText: string; normalizedText: string; category: EntityCategory; pseudonymId: number; restorePreference: boolean; createdAt: number; lastSeenAt: number; }`

**src/ai/sanitization/regex-patterns.ts** — Structured PII regex library:
- Export `PII_PATTERNS` array with regexes for: email, US phone, international phone, credit card (16 digits with optional separators), IBAN, dollar amounts, API key prefixes (sk-, pk-, api-, key-, token-, secret-, bearer-), GitHub tokens (ghp_, gho_, ghu_, ghs_, ghr_), Anthropic keys (sk-ant-), OpenAI keys (sk-), SSN pattern (XXX-XX-XXXX)
- Each pattern: `{ category: EntityCategory; pattern: RegExp; name: string }`
- Export `detectWithRegex(text: string): DetectedEntity[]` — runs all patterns against text, returns matched entities with start/end offsets, confidence: 1.0, source: 'regex'
- Handle global regex correctly: create new RegExp or use matchAll to avoid stateful lastIndex issues
- Pure function, no side effects

**src/ai/sanitization/entity-registry.ts** — Persistent entity-to-pseudonym registry:
- Import db from storage/db.ts and EntityRegistryEntry, EntityCategory from types.ts
- Pure module — functions take db instance or are free functions operating on the Dexie table
- `getOrCreatePseudonym(realText: string, category: EntityCategory): Promise<{ pseudonymId: number; tag: string }>` — Looks up normalizedText (lowercase trimmed) + category in entityRegistry table. If found, updates lastSeenAt and returns existing pseudonymId. If not found, generates next pseudonymId for that category (max existing + 1, or 1) and inserts new entry. Returns tag like `<Person 12>` with capitalized category name.
- `buildEntityMap(entities: DetectedEntity[]): Promise<{ entityMap: Map<string, string>; reverseMap: Map<string, string> }>` — For each entity, get or create pseudonym. Build bidirectional maps: pseudonym tag -> real text and real text -> pseudonym tag. Respects restore preferences: if restorePreference is true for an entity, exclude it from the maps (it stays unredacted).
- `getRestorePreference(realText: string, category: EntityCategory): Promise<boolean>` — lookup in registry
- `setRestorePreference(realText: string, category: EntityCategory, restore: boolean): Promise<void>` — update registry entry

**src/workers/sanitization-worker.ts** — Dedicated web worker for NER inference:
- Follow the pattern of src/search/embedding-worker.ts exactly:
  - `import { pipeline, env } from '@huggingface/transformers';`
  - `env.allowRemoteModels = false; env.allowLocalModels = true; env.localModelPath = '/models/';`
  - Configure ORT: `import * as ort from 'onnxruntime-web'; ort.env.wasm.proxy = false; ort.env.wasm.numThreads = 1;`
- NER pipeline singleton with lazy loading:
  - `pipeline('token-classification', 'sanitization/sanitize-check', { dtype: 'q8' })`
  - Use `aggregation_strategy: 'simple'` for merged entity spans (Pitfall 6 from RESEARCH.md)
- Message protocol (follows existing worker convention):
  - Incoming: `{ type: 'SANITIZE'; id: string; text: string }` and `{ type: 'LOAD_NER' }`
  - Outgoing: `{ type: 'SANITIZE_RESULT'; id: string; entities: Array<{ text: string; category: string; start: number; end: number; confidence: number }> }`
  - Outgoing: `{ type: 'SANITIZE_ERROR'; id: string; error: string }`
  - Outgoing: `{ type: 'NER_READY' }`, `{ type: 'NER_LOADING' }`, `{ type: 'NER_ERROR'; error: string }`
- Map entity_group labels from model output (e.g., "B-PERSON", "PERSON") to EntityCategory strings
- Graceful degradation: if model fails to load, SANITIZE_ERROR returned — caller falls back to regex-only
- DO NOT eagerly load NER model at worker init — lazy-load on first SANITIZE message or explicit LOAD_NER. This keeps memory footprint zero until cloud is actually used.

**src/ai/sanitization/sanitizer.ts** — Core sanitization pipeline (PURE MODULE — no store imports):
- Create and manage the sanitization worker instance (lazy — created on first call)
- `initSanitizationWorker(): void` — creates Worker if not already created, sends LOAD_NER
- `detectEntities(text: string): Promise<DetectedEntity[]>` — Runs NER (via worker postMessage) and regex (via detectWithRegex from regex-patterns.ts) in parallel using Promise.all. Merges results with overlap resolution: when NER and regex spans overlap, prefer the longer/more-specific match; regex CONTACT takes precedence over NER PERSON for substrings within email/phone patterns (Pitfall 3 from RESEARCH.md). Returns deduplicated entity list sorted by start offset.
- Worker communication: use a pending-request Map<string, { resolve, reject }> pattern with unique IDs (crypto.randomUUID or counter) for request-response correlation. Set up worker.onmessage handler on init.
- `sanitizeText(text: string): Promise<SanitizedResult>` — Calls detectEntities, then buildEntityMap from entity-registry, then replaces entities in text with pseudonym tags (replace from end to start to preserve offsets). Creates SanitizedPrompt via createSanitizedPrompt. Returns SanitizedResult with prompt, entities, entityMap, reverseMap.
- `dePseudonymize(response: string, reverseMap: Map<string, string>): string` — Replaces `<Person N>`, `<Location N>`, `<Financial N>`, `<Contact N>`, `<Credential N>` patterns in the response string with real values from the reverseMap (note: reverseMap for de-pseudonymization is actually entityMap which maps pseudonym -> real). Use regex: `/<(Person|Location|Financial|Contact|Credential)\s+\d+>/g`
- `disposeSanitizationWorker(): void` — terminates worker, clears singleton
- If worker is not available (model failed to load), fall back to regex-only detection — still produces SanitizedPrompt
