Create `scripts/train/20_generate_gtd_data.py` following the pattern from `10_generate_sanitization_data.py` (Faker templates + argparse).

The script accepts `--classifier {gtd-routing|actionability|project-detection|context-tagging}` and `--count N` (default 1000, meaning 1000 examples per label).

For each classifier, define template lists with Faker placeholders:

**GTD Routing (4-way: next-action, waiting-for, someday-maybe, reference):**
- next-action: Clear imperative actions ("Call {person} about {topic}", "Buy {item} from {store}", "Email {person} the {document}", "Fix the {problem} in the {location}", "Review {document} and send feedback", "Schedule {event} with {person}", "Update {system} with new {data}", "Submit {document} to {person}")
- waiting-for: Delegated/external dependency ("Waiting for {person} to send {document}", "{person} will get back to me about {topic}", "Order placed for {item}, tracking {number}", "Submitted request to {department} for {thing}", "{person} is reviewing the {document}", "Package from {company} arriving {date}")
- someday-maybe: Aspirational/uncommitted ("Maybe learn {skill} someday", "Consider switching to {tool} for {purpose}", "Would be nice to visit {location}", "Look into {topic} when there's time", "Might want to try {activity}", "Think about {idea} for the future")
- reference: Informational/no action ("{person}'s phone number is {phone}", "The {system} password is stored in {tool}", "Office hours are {time} to {time}", "{topic} documentation is at {url}", "Company policy on {topic}: {summary}", "Meeting notes from {date}: {summary}")
- Include 15-20% ambiguous borderline examples per CONTEXT.md decision. Label by most GTD-orthodox interpretation.

**Actionability (binary: actionable, non-actionable):**
- actionable: Has concrete physical next step ("Call {person}", "Buy {item}", "Send {document}", "Fix {problem}", "Schedule {event}", "Write {document}", "Clean {location}", "Pay {amount} for {thing}")
- non-actionable: Informational/reference/observation ("{person}'s birthday is {date}", "The meeting was productive", "Revenue was up {percent}", "The new policy takes effect {date}", "{topic} is interesting", "Server uptime: {percent}")
- Ambiguous cases: "{item} is broken" (could imply action), "Need more {resource}" (vague), "{person} mentioned the {topic}" (could imply follow-up)

**Project Detection (binary: project, single-action):**
- project: Multi-step outcomes ("Plan the {event}", "Organize {event} for {group}", "Renovate the {room}", "Launch {product}", "Migrate {system} to {platform}", "Hire a new {role}", "Set up {system} for the team")
- single-action: Atomic tasks ("Call {person}", "Buy {item}", "Send {document} to {person}", "Book {resource} for {date}", "Print {document}", "Update {field} in {system}", "Reply to {person}'s email")
- Ambiguous: "Research {topic}" (could be single Google search or multi-week project)

**Context Tagging (6-way: @computer, @phone, @errands, @home, @office, @agenda):**
- @computer: Digital work ("Update the {document} in {tool}", "Deploy {system} to {environment}", "Write code for {feature}", "Review pull request from {person}", "Research {topic} online", "Export {data} to {format}")
- @phone: Calls/texts ("Call {person} about {topic}", "Text {person} the {info}", "Leave voicemail for {person}", "Dial into the {meeting} conference call", "Call {company} support about {issue}")
- @errands: Out-and-about ("Pick up {item} from {store}", "Drop off {item} at {location}", "Return {item} to {store}", "Go to {location} for {purpose}", "Mail {item} at the post office", "Get {item} from the pharmacy")
- @home: Domestic ("Clean the {room}", "Fix the {fixture}", "Mow the lawn", "Cook {meal} for dinner", "Organize the {space}", "Water the plants", "Do laundry")
- @office: In-person workplace ("Set up the {room} for the meeting", "Print {document} on the office printer", "Restock the {supply}", "Check the mail in the office mailbox", "Talk to {person} at their desk")
- @agenda: Discuss with specific person ("Talk to {person} about {topic}", "Ask {person} for input on {project}", "Discuss {topic} with {person} at next 1:1", "Bring up {topic} with {person}", "Get {person}'s approval on {document}")
- Balance templates deliberately across all 6 contexts to avoid @computer overrepresentation (RESEARCH.md Pitfall 5). Use domain-specific verbs for each context.

Each template should use Faker generators for variable slots: `fake.name()`, `fake.company()`, `fake.catch_phrase()`, `fake.job()`, `fake.city()`, `fake.date_this_year()`, etc. Use simple {placeholder} → Faker replacement pattern (NOT BIO tagging like sanitization -- this is classification, not NER).

Output format: one JSON object per line in JSONL: `{"text": "Call John about the budget", "label": "next-action"}`

The script should:
1. Generate `count` examples per label (not `count` total)
2. Shuffle all examples
3. Write to `scripts/training-data/{classifier-name}.jsonl`
4. Print statistics: total samples, per-label counts, sample preview
5. Set `Faker.seed(42)` and `random.seed(42)` for reproducibility

Use `-u` flag reminder in docstring (Python buffering issue from MEMORY.md).

**Part A: Create `scripts/train/21_train_gtd_classifier.py`**

Follow the pattern from `03_train_classifier.py` but self-contained (embeds + trains + exports in one script). Accepts `--classifier {gtd-routing|actionability|project-detection|context-tagging}`.

Per-classifier config (from CONTEXT.md locked decisions):
```python
CLASSIFIER_CONFIGS = {
    'gtd-routing': {
        'hidden_layers': (256, 128),
        'confidence_threshold': 0.70,
        'input_file': 'gtd-routing.jsonl',
        'output_model': 'gtd-routing.onnx',
        'output_classes': 'gtd-routing-classes.json',
    },
    'actionability': {
        'hidden_layers': (128, 64),
        'confidence_threshold': 0.80,
        'input_file': 'actionability.jsonl',
        'output_model': 'actionability.onnx',
        'output_classes': 'actionability-classes.json',
    },
    'project-detection': {
        'hidden_layers': (128, 64),
        'confidence_threshold': 0.75,
        'input_file': 'project-detection.jsonl',
        'output_model': 'project-detection.onnx',
        'output_classes': 'project-detection-classes.json',
    },
    'context-tagging': {
        'hidden_layers': (256, 128),
        'confidence_threshold': 0.65,
        'input_file': 'context-tagging.jsonl',
        'output_model': 'context-tagging.onnx',
        'output_classes': 'context-tagging-classes.json',
    },
}
```

Pipeline steps (for each classifier):
1. Load JSONL training data from `scripts/training-data/{input_file}`
2. Embed all texts via sentence-transformers MiniLM (`normalize_embeddings=True` for browser parity -- RESEARCH.md Pitfall 4)
3. Train/test split (80/20, stratified, random_state=42)
4. Train MLPClassifier with config hidden_layers, activation="relu", max_iter=500, random_state=42, early_stopping=True
5. Wrap with CalibratedClassifierCV(method="sigmoid", cv=5) for Platt calibration
6. Evaluate: print classification_report, confusion_matrix, calibration stats at classifier's threshold
7. Export to ONNX: target_opset=17, options={"zipmap": False} (WASM-compatible -- DO NOT change)
8. Write class mapping JSON to public/models/classifiers/
9. Generate validation artifacts: test_embeddings, python_predictions, python_probabilities -- saved to `scripts/train/gtd_{classifier_name}_test_embeddings.json` etc.
10. Print summary with accuracy, calibration, file sizes

**Part B: Create `scripts/train/22_validate_gtd_models.mjs`**

Follow the pattern from `04_validate_model.mjs`. Accepts `--classifier <name>` or `--all`.

For each classifier:
1. Load ONNX model via onnxruntime-node
2. Load test embeddings + Python predictions from Part A
3. Run Node.js ONNX inference on test embeddings
4. Compare Node.js top-1 predictions to Python top-1 predictions
5. Require >95% match rate (hard pass/fail gate)
6. Print per-classifier results and overall summary
