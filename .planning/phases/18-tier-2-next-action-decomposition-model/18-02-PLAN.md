**categories.ts** — Define types and all ~35 decomposition templates:

```typescript
export interface TemplateStep {
  template: string;        // e.g., "Research {topic} options"
  defaultType: AtomType;   // AI-suggested atom type for this step
  slots: string[];         // Which slots this step needs: ['topic']
}

export interface DecompositionTemplate {
  category: string;
  applicableTo: ('task' | 'decision')[];
  steps: TemplateStep[];
}

export interface DecomposedStep {
  text: string;            // Filled template text
  suggestedType: AtomType; // From template defaultType
  stepIndex: number;       // 0-based position in template
}

export interface DecompositionResult {
  category: string;
  confidence: number;
  steps: DecomposedStep[];
  originalText: string;
}
```

Define DECOMPOSITION_CATEGORIES as Record<string, DecompositionTemplate> with all ~35 categories. Each category has 3-5 steps. Steps must be GTD-style "next physical actions" — concrete, verb-first, immediately doable.

The category names MUST match the labels in decomposition-classes.json produced by the training script (Plan 01). Reference the training data generator to ensure alignment.

**slot-extractor.ts** — Extract entities from input text for template filling:

```typescript
export interface ExtractedSlots {
  topic: string;       // Primary subject (always filled — fallback to full text)
  person?: string;     // Person name from regex
  location?: string;   // Place from regex
  item?: string;       // Object referenced
  event?: string;      // Event name
}
```

Extraction strategy:
1. Import and use detectWithRegex from sanitization/regex-patterns.ts for PERSON and LOCATION entities
2. For {topic}: strip common leading verbs (plan, buy, organize, research, decide on, choose, find, get, make, set up, create, fix, schedule, arrange, prepare, complete, start, learn, update, write, send, clean, review, cancel, renew) and prepositions from input text. The remaining noun phrase is the topic. If extraction produces empty string, use full input text.
3. For {item}: look for patterns like "new {item}", "a {item}", "the {item}" after verb stripping
4. For {person}: use PERSON entities from detectWithRegex
5. For {location}: use LOCATION entities from detectWithRegex
6. All optional slots default to undefined if not found

Pure module — NO store imports.

**decomposer.ts** — Main decomposition pipeline:

```typescript
export async function decomposeAtom(
  text: string,
  atomType: 'task' | 'decision',
  classifyFn: (text: string) => Promise<{ category: string; confidence: number }>,
): Promise<DecompositionResult>
```

Pipeline:
1. Call classifyFn to get pattern category + confidence
2. Look up category in DECOMPOSITION_CATEGORIES
3. Filter templates by applicableTo matching atomType
4. If no match or confidence below 0.60 threshold, use a generic fallback template:
   - For tasks: ["Clarify what done looks like for {topic}", "Identify the very next physical action for {topic}", "Do the next action or schedule it"]
   - For decisions: ["Research options for {topic}", "Define criteria for {topic}", "Compare top options for {topic}", "Make and record decision on {topic}"]
5. Extract slots from input text via extractSlots()
6. Fill template steps with extracted slots. For any slot reference like {person} where slot is undefined, remove the slot reference and surrounding whitespace cleanly (e.g., "Meet with {person} about {topic}" with no person -> "Meet about {topic}")
7. Return DecompositionResult with filled steps

Pure module — NO store imports. classifyFn injected by caller (tier2 handler).
