
Types:
```typescript
interface DecompositionFlowState {
  steps: DecomposedStep[];
  currentStepIndex: number;
  acceptedSteps: AcceptedStep[];
  originalText: string;
  originalAtomId: string;
  category: string;
  confidence: number;
  phase: 'stepping' | 'project-prompt' | 'done';
}

interface AcceptedStep {
  text: string;
  type: AtomType;
  edited: boolean;
}
```

Export a `startDecomposition` function that:
1. Takes `(atomId: string, text: string, atomType: 'task' | 'decision')` as parameters
2. Calls `dispatchTiered({ task: 'decompose', features: { content: text, atomType } })` to get classification + steps
3. If result has decomposition steps, sets decompositionState and shows the flow
4. If no steps (low confidence fallback), still shows the generic decomposition steps

Component renders:
- **Backdrop** (click outside to close, same pattern as AIQuestionFlow)
- **Step card** showing:
  - Header: "Step {n} of {total}" with the category name
  - Step text (editable — use a text input pre-filled with the step text)
  - Type selector: small inline buttons for the step's atom type (task, decision, fact, insight, event) — pre-selected to suggestedType, user can override per CONTEXT.md
  - Three action buttons: "Accept" (primary), "Skip" (secondary), and a small edit toggle
  - When user clicks Accept: save step to acceptedSteps array, advance to next step
  - When user clicks Skip: advance to next step without saving
- After last step, transition to **project prompt phase**:
  - "Mark '{originalText}' as a project?" with Yes/No buttons
  - Per CONTEXT.md: user decides each time (not auto-marked)
- After project prompt (or skip), transition to **done phase**:
  - Create atoms for all accepted steps via `sendCommand({ type: 'CREATE_INBOX_ITEM', payload: { content: step.text } })` — steps go to inbox where the type classifier will classify them, OR use CLASSIFY_INBOX_ITEM if we have section routing. Since steps are new content, CREATE_INBOX_ITEM is simplest — they'll enter the normal triage flow.
  - Actually, per CONTEXT.md: "Each decomposed step gets section assignment via the section routing classifier (AI suggests per step, not inherited from parent)." So create as inbox items and let the existing triage pipeline handle them: `sendCommand({ type: 'CREATE_INBOX_ITEM', payload: { content: step.text } })`
  - If user said yes to project: mark the parent atom's type as 'task' with a note that it's a project (the isProject metadata is on TriageSuggestion, not on the atom itself — so just display a brief confirmation)
  - Close the flow

Keyboard support:
- Escape closes the flow
- Enter accepts the current step
- Tab skips the current step
