Purpose: These models enable offline sub-second detection of vague/incomplete atoms and identification of which specific information gaps exist, forming the ML foundation for the clarification wizard.

Output: 6 trained ONNX models in public/models/classifiers/, 6 JSONL training datasets, and 3 training scripts (generate, train, validate).
</objective>

  <action>
Create `scripts/train/40_generate_clarification_data.py` following the Phase 17/18 Faker-based generation pattern (reference `20_generate_gtd_data.py` and `30_generate_decomposition_data.py`).

Generate training data for 6 classifiers:

**Completeness gate** (`clarification-completeness.jsonl`):
- 1000 "complete" examples: well-specified atoms with clear outcomes, next actions, timeframes, and context. Cover all 5 atom types (task, event, fact, decision, insight). Examples: "Call dentist at 555-1234 tomorrow at 2pm to schedule cleaning", "Decision: Switch to AWS us-east-1 for lower latency by March 15 — notify DevOps team"
- 1000 "incomplete" examples: vague atoms missing key GTD info. Examples: "dentist", "fix the thing", "meeting about stuff", "need to do that project"
- 200 ambiguous borderline examples (labeled by most GTD-orthodox interpretation)
- CRITICAL: Include ~200 "complete" examples that contain enrichment format (`\n---\nOutcome: ...\nDeadline: ...`) so the completeness gate learns enriched text = complete. This prevents re-triage infinite loops (Research Pitfall 5).

**5 binary missing-info classifiers** (each produces `clarification-missing-{category}.jsonl`):
- Per category, 1000 "missing" + 1000 "not-missing" + ~200 ambiguous = ~2200 per file
- **missing-outcome**: "missing" = atoms with no clear desired end state; "not-missing" = atoms with explicit outcomes
- **missing-next-action**: "missing" = atoms with no concrete physical next step; "not-missing" = atoms with verb-first actions
- **missing-timeframe**: "missing" = atoms with no temporal reference; "not-missing" = atoms with dates, "by Friday", "this week", etc.
- **missing-context**: "missing" = atoms with no who/where/what-tool; "not-missing" = atoms with @context, person names, locations
- **missing-reference**: "missing" = atoms with no project/area link; "not-missing" = atoms mentioning projects, areas, or categories

Use Faker for realistic variety: names, companies, dates, locations, tools, projects. Each category's data should span all 5 atom types. Total output: ~20,000+ examples across 6 JSONL files.

Each JSONL line: `{"text": "...", "label": "complete|incomplete" or "missing|not-missing"}`.

Run with: `python -u scripts/train/40_generate_clarification_data.py`

Pipeline per classifier (same as Phase 17/18):
1. Load JSONL training data
2. Embed with sentence-transformers MiniLM (normalize_embeddings=True)
3. Stratified train/test split (80/20)
4. Train sklearn MLPClassifier with Platt calibration (probability=True)
5. Evaluate: print accuracy, classification report. Assert >95% accuracy.
6. Export to ONNX via skl2onnx (float input [None, 384])
7. Save classes JSON and validation artifacts (test embeddings, predictions, probabilities)

CLI: `python -u scripts/train/41_train_clarification_classifier.py --classifier completeness-gate` or `--classifier all`.
