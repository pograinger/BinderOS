  <action>
Create `src/ui/components/ClarificationFlow.tsx` following the DecompositionFlow pattern exactly (module-level signals, modal overlay, step-by-step).

**Module-level signals:**
- `clarificationActive: boolean` — controls modal visibility
- `clarificationAtom: { id, title, content, type } | null` — the atom being clarified
- `clarificationQuestions: ClarificationQuestion[]` — questions to present (from binary classifiers + binder config)
- `currentQuestionIndex: number` — which question is showing
- `answers: ClarificationAnswer[]` — accumulated answers
- `showSummary: boolean` — whether to show summary screen
- `cloudOptions: Map<MissingInfoCategory, string[] | null>` — cloud-enhanced options (when available)

**Exported functions:**
- `startClarification(atom: { id, title, content, type }, missingCategories: MissingInfoCategory[]): void`
  - Sets module signals, generates template questions via `generateTemplateOptions()` for each missing category
  - Orders questions per binder config `categoryOrdering`
  - Optionally kicks off cloud option prefetch if cloud is available
- `closeClarification(): void` — cleans up signals, aborts any pending cloud requests

**Component structure:**
1. **Modal overlay** — backdrop click/Escape to close (applies partial answers on close)
2. **Header** — Original atom title/content pinned at top for reference
3. **Question view** (when `!showSummary`):
   - Subtle category indicator next to question (e.g., "outcome", "timeframe") — styled as a small badge
   - Question text from binder config
   - 3-4 option buttons (from templates, upgraded to cloud options when available)
   - Freeform text input below options
   - "Skip" button
   - On option select or freeform submit: record answer, advance to next question
   - On skip: record as skipped, advance to next question
   - If cloud options arrive (via prefetch promise), replace template options with cloud options (guard against stale updates if question already advanced)
4. **Summary view** (after last question or when user closes):
   - "Added: Outcome, Deadline, Context" listing what was answered
   - Preview of enriched content
   - "Done" button to close

**On close (or Done):**
1. Call `appendEnrichment(originalContent, answers)` to build enriched content
2. Emit result via a callback (set during startClarification): `onComplete(ClarificationResult)`
3. Clean up cloud prefetch AbortControllers
4. Close modal

**Styling (`ClarificationFlow.css`):**
- Follow DecompositionFlow.css patterns for modal overlay, backdrop, card
- Category badge: small pill with muted color
- Option buttons: similar to AIQuestionFlow button style
- Freeform input: bordered text input below options
- Summary section: compact key-value display with muted labels
