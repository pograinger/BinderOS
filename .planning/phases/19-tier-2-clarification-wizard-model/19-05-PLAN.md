**Create `src/ai/clarification/option-ranking.ts`** — pure module for self-learning:

`rankOptions(category: MissingInfoCategory, defaultOptions: string[], history: ClassificationEvent[]): string[]`
- Filter history to clarification events for this category
- Count frequency of each `optionSelected` value
- Sort default options by frequency (most-selected first)
- Options not in history keep their default order but appear after ranked ones
- Freeform entries that appear 3+ times get promoted to the option list (manual retraining: included in next correction JSONL export, not auto-promoted as a permanent option)
- Cold start (no history): return default options in original order

`getSkipPatterns(history: ClassificationEvent[]): Record<MissingInfoCategory, number>`
- Count how many times each category was skipped (optionSelected === null && wasFreeform === false)
- Return skip count per category

`shouldDeprioritizeCategory(category: MissingInfoCategory, skipPatterns: Record<MissingInfoCategory, number>, totalClarifications: number): boolean`
- If skip rate for this category exceeds 70% (and totalClarifications > 5), deprioritize
- Deprioritized categories move to end of question ordering (still presented, just last)

**Integrate into `src/ui/components/ClarificationFlow.tsx`:**
- In `startClarification()`, load classification history once via `getClassificationHistory()`
- Apply `rankOptions()` to each question's options before rendering
- Apply `shouldDeprioritizeCategory()` to reorder questions (move deprioritized to end)
- Cloud options, when they arrive, are NOT ranked (they're already contextual) — only template options get ranked

**Extend `src/ui/signals/store.ts`** (minor):
- In `handleClarificationComplete()`, ensure freeform answers are included in the classification log event's `freeformText` field so ranking can detect frequently-typed freeform entries
