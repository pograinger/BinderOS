Create `scripts/train/classifier_registry.py` as a shared module imported by all 50-53 scripts. It must contain:

1. `CLASSIFIER_REGISTRY` dict mapping 12 classifier names to their metadata:
   - `jsonl`: filename in scripts/training-data/ (e.g., "type-classification.jsonl")
   - `train_script`: command to retrain (e.g., "03_train_classifier.py" for type, "21_train_gtd_classifier.py --classifier gtd-routing" for GTD classifiers, "31_train_decomposition_classifier.py" for decomposition, "41_train_clarification_classifier.py --classifier completeness-gate" for clarification classifiers)

   - `onnx_model`: filename in public/models/classifiers/ (e.g., "triage-type.onnx")
   - `classes_json`: filename for class mapping (e.g., "triage-type-classes.json")
   - `label_field`: "label" for all classifiers
   - `hidden_layers`: tuple matching architecture (e.g., (256,128) for multi-class, (128,64) for binary)
   - `is_multi_class`: bool (True for type/gtd-routing/context-tagging/decomposition, False for binary classifiers)
   - `class_names`: list of valid label strings per classifier
   - `gtd_definitions`: brief GTD definition string per classifier explaining what each class means (for use in expert exam prompts)

2. Helper constants: `REPO_ROOT`, `TRAINING_DATA_DIR`, `CLASSIFIER_DIR`, `EXPERT_EXAM_DIR`, `REPORTS_DIR`

Then create `scripts/train/50_benchmark_models.py`:

**CLI interface:**
- `--classifier {name|all}` -- which classifier(s) to benchmark (default: all)
- `--expert-exam` -- also generate and score a cloud expert exam test set
- `--model {haiku|sonnet}` -- Claude model for expert exam generation (default: haiku)
- `--exam-count N` -- examples per class for expert exam (default: 50; for binary classifiers use 100)

**Benchmark mode (default):**
1. For each selected classifier:
   a. Load the JSONL training data file
   b. Embed all examples using sentence-transformers MiniLM-L6-v2
   c. Split 80/20 stratified train/test (use random_state=42 for reproducibility)
   d. Load the existing ONNX model via onnxruntime
   e. Run inference on the test split
   f. Compute per-class precision, recall, F1 via sklearn.metrics.classification_report(output_dict=True)
   g. Identify weakest class (lowest F1) and overall weighted accuracy
   h. Record results in a structured dict

2. Write a Markdown report to `scripts/train/reports/benchmark_{YYYYMMDD_HHMMSS}.md` with:
   - Summary table: classifier name, overall accuracy, weakest class, weakest class F1
   - Per-classifier section with full classification_report table and confusion matrix
   - Low-confidence examples section: list 10 examples per classifier with confidence < threshold (helps downstream scripts target weak spots)

3. Also write a machine-readable JSON to `scripts/train/reports/benchmark_{timestamp}.json` containing the full results dict (downstream scripts 51-53 will read this to know where to focus).

**Expert exam mode (--expert-exam):**
1. For each selected classifier:
   a. Use the Anthropic API to generate `exam-count` examples per class
   b. Prompt Claude as a GTD expert to create realistic, messy, natural-language examples that test GTD boundaries -- NOT Faker-style templates. Use the `gtd_definitions` from the registry. Include deep GTD methodology references (2-minute rule, horizons of focus, natural planning model, someday/maybe boundaries, weekly review criteria).
   c. Generate in difficulty tiers: 25% easy, 25% medium, 25% hard, 25% adversarial
   d. Use structured output (output_config.format with json_schema) following 01_generate_data.py pattern
   e. Include `reasoning` and `difficulty` fields in the schema for quality verification
   f. Write expert exam to `scripts/training-data/expert-exam/{classifier_name}.jsonl`
   g. Deduplicate against existing training data (exact text match, case-insensitive)

2. Score all classifiers against the expert exam:
   a. Embed expert exam examples with MiniLM
   b. Run ONNX inference
   c. Compute accuracy and per-class metrics
   d. Append expert exam results to the benchmark report

3. Rate limiting: 0.05s sleep between API calls, exponential backoff on RateLimitError (matching 01_generate_data.py pattern).

Use `python -u` compatible output (print with flush=True or use tqdm progress bars). Follow the established pattern: dotenv for ANTHROPIC_API_KEY, Path-based file handling, argparse CLI.

For the type classifier specifically, include the augmentation categories from memory notes in the expert exam prompts:
- Multi-step/compound tasks ("Call dentist and then update insurance info")
- Ambiguous borderline facts that look like tasks ("The faucet is leaking")
- Vague/incomplete events ("Dentist next week")
- Implicit decisions ("We're going with the new vendor")
- Negative/frustrated-tone insights ("I always underestimate how long things take")

Create `scripts/train/reports/.gitkeep` and `scripts/training-data/expert-exam/.gitkeep` as empty files to ensure directories are tracked.
