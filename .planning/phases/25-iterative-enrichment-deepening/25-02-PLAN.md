    1. **Extend createEnrichmentSession params**: Add optional fields:
       - `depthMap?: Record<string, number>` -- existing per-category depths (from InboxItem.enrichmentDepth)
       - `cognitiveSignals?: SignalVector | null` -- cached cognitive signals

    2. **Replace category-skip logic**: Instead of filtering out answered categories entirely:
       - For each of the 5 categories, check if it has a prior answer in `allEnrichments`
       - If NO prior answer: generate first-pass question via `generateTemplateOptions` (existing)
       - If HAS prior answer AND `depthMap[cat] < MAX_ENRICHMENT_DEPTH`: generate follow-up via `generateFollowUpOptions` (new from Plan 01), passing `priorAnswer`, `depth+1`
       - If HAS prior answer AND `depthMap[cat] >= MAX_ENRICHMENT_DEPTH`: skip (maxed out)

    3. **Add signal-guided priority ordering**: After generating all candidate questions, if `cognitiveSignals` is provided (not null), reorder by signal relevance:
       - Define a static `SIGNAL_CATEGORY_MAP` mapping `CognitiveModelId -> MissingInfoCategory[]` (from research):
         - `priority-matrix` -> `['missing-outcome', 'missing-timeframe']`
         - `collaboration-type` -> `['missing-context', 'missing-reference']`
         - `cognitive-load` -> `['missing-next-action']`
         - `gtd-horizon` -> `['missing-outcome']`
         - `time-estimate` -> `['missing-timeframe']`
         - `energy-level` -> `['missing-context']`
         - `knowledge-domain` -> `['missing-reference']`
       - For each category, compute a relevance score: sum of `(1 - confidence)` for signals that map to this category. Higher = more uncertain = more valuable to ask.
       - Sort questions by relevance (descending), with first-pass questions always before follow-ups when relevance is equal.
       - If signals is null, keep default GTD ordering (outcome, next-action, timeframe, context, reference).

    4. **Initialize new session fields**:
       - `categoryDepth: depthMap ?? {}`
       - `cognitiveSignals: cognitiveSignals ?? null`
       - `activeDeepening: null`

    5. **Update applyAnswer**:
       - After recording the answer, increment `categoryDepth` for the answered category: `categoryDepth[answer.category] = (categoryDepth[answer.category] ?? 0) + 1`
       - For follow-up answers (category already in answers), replace the existing answer for that category instead of appending a duplicate. Check if `session.answers` already has an entry for `answer.category` -- if so, replace it; if not, append.

    6. **Keep advanceSession unchanged**: The phase state machine stays the same. Depth affects question GENERATION, not session TRANSITIONS (per research anti-pattern guidance).

The changes must be backward-compatible -- calling createEnrichmentSession without depthMap/cognitiveSignals behaves identically to current behavior (answered categories still skipped when no depthMap provided, which means depthMap defaults to {} and all answered categories have depth 0 < MAX_DEPTH, so they will get follow-ups). To maintain backward compat for existing callers that expect "skip answered": when `depthMap` is not provided, treat answered categories as having depth `MAX_ENRICHMENT_DEPTH` (maxed out) so they are skipped. Only when `depthMap` is explicitly provided does iterative deepening activate.
