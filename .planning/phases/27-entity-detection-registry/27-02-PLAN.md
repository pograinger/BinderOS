    **Step 1: Create src/ui/components/EntityBadges.tsx.**
    - SolidJS component with props: `{ mentions: EntityMention[]; maxVisible?: number }`.
    - Color map by entity type using Tailwind classes:
      - PER: `bg-blue-100 text-blue-800 dark:bg-blue-900 dark:text-blue-200`
      - ORG: `bg-amber-100 text-amber-800 dark:bg-amber-900 dark:text-amber-200`
      - LOC: `bg-green-100 text-green-800 dark:bg-green-900 dark:text-green-200`
      - MISC: `bg-gray-100 text-gray-700 dark:bg-gray-800 dark:text-gray-300`
      - DATE: `bg-purple-100 text-purple-800 dark:bg-purple-900 dark:text-purple-200`
    - Sort by confidence descending, deduplicate by entityText (case-insensitive, keep highest confidence).
    - Show MISC badges but NOT DATE badges (per Claude's discretion -- MISC can be useful, DATE are rarely identity-meaningful).
    - Show top 5 (or maxVisible prop), with "+N more" overflow chip if more exist.
    - Use `<Show when={sorted().length > 0}>` to hide entire section when empty.
    - Chip styling: `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium`.
    - Tap behavior (Claude's discretion): simple expand to show all badges when "+N more" is clicked. No navigation.
    - Container: `flex flex-wrap gap-1.5 mt-2` -- same zone as tags, below content.

    **Step 2: Integrate into AtomDetailView.tsx.**
    - Import `EntityBadges` from `../components/EntityBadges`.
    - Import `getIntelligence` from `../../storage/atom-intelligence`.
    - Import `EntityMention` type from `../../types/intelligence`.
    - Use `createResource` or `createSignal` + `createEffect` to load entity mentions from the sidecar:
      - Key on `state.selectedAtomId` -- when it changes, reload sidecar data.
      - Call `getIntelligence(atomId)` to get sidecar row.
      - Extract `entityMentions` array (default to empty array if no sidecar row).
    - CRITICAL (Pitfall 4): Use `createResource` keyed on selectedAtomId, NOT SolidJS store, for sidecar data. The sidecar is loaded from Dexie asynchronously and is not part of the reactive store.
    - Render `<EntityBadges mentions={entityMentions()} />` below the content textarea and above the tags section.
    - Wrap in `<Show when={entityMentions().length > 0}>` for clean empty state.
    - The badges should appear silently when entity detection completes. If the user is already viewing the atom when detection finishes, badges will appear on next sidecar reload. A simple polling or signal could be added, but for Phase 27 scope, badges appearing on view open is sufficient.
    - To handle the case where detection completes while viewing: add a reactive trigger. In the detection orchestrator (entity-detector.ts from Plan 01), after writeEntityMentions completes, it could increment a global signal. AtomDetailView watches this signal to refetch sidecar data. OR, keep it simple: badges load when detail view opens. User can close and reopen to see badges if they just created the atom. This is acceptable per the "silent badge appearance" decision.
