<decisions_made>

- **entityTextFilter over proximity scoring**: For healthcare-title-dr, filtering by entity text prefix is simpler and more reliable than word-proximity scoring. Works because doctors are consistently named "Dr. X".
- **Post-processing suppression over pattern ordering**: `cleanSuppressedRelations()` runs after all patterns fire, avoiding corpus-ordering dependency. More robust than checking during upsert.
- **Role-word merging over alias approach**: Adding role words ("mom", "son") as entity aliases caused false substring matches (e.g., "son" matched "Morri**son**"). Direct entity merging via `mergeEntities()` avoids this.
- **Hand-crafted corpora over API-generated**: Better control over edge cases, no API key dependency, targeted pattern coverage.
- **Generalized bystander problem**: The core issue across ALL pattern categories is that sentence-scoped matching tags every PER entity in a sentence, not just the one the keyword refers to. The healthcare-title fix was specific; the next round needs a general solution.
</decisions_made>

<context>
The bystander problem is THE central precision issue. When a sentence like "Priya is taking Arjun to his dentist appointment, Sunita will pick up Meera from school" contains 4 PER entities, both "dentist" and "school" keywords fire and tag ALL 4 entities with healthcare-provider AND child relationships. The entityTextFilter fix works for "Dr." keywords (filter by entity name prefix) but doesn't generalize to role keywords like "dentist", "school", etc.

Possible general solutions to explore:
1. **Keyword-entity proximity**: Only fire when keyword is within N words of entity mention
2. **Subject/object parsing**: Use simple heuristics (possessive "his dentist", "my son") to determine which entity the keyword refers to
3. **Negative entity filters**: For child-context, exclude entities that already have non-child relationships (adults)
4. **Confidence dampening**: When multiple entities in a sentence, divide confidence by entity count (so 4 entities = 0.25x confidence each)

Option 1 (proximity) is probably the best balance of simplicity and effectiveness.
</context>
