<objective>
Build the headless cognitive harness that validates the full local pipeline against a synthetic user profile, measuring how well the relationship inference engine learns entity relationships from progressive atom processing.

Purpose: The harness proves the v5.0 thesis — that the local cognitive stack can learn meaningful facts about the user (entities, relationships) without cloud assistance. It measures the learning curve as atoms accumulate, reports precision/recall at checkpoints, and computes a privacy score showing how entity knowledge enables semantic sanitization. This infrastructure is the foundation for Phase 29's adversarial training loop.

Output: Synthetic user profile, pre-generated corpus, headless pipeline runner, scoring engine, JSON + Markdown reports in `scripts/harness/reports/`.
</objective>

Create `scripts/harness/score-graph.ts` — scoring logic that compares detected graph against ground truth:

**Scoring methodology:**
- `scoreEntityGraph(store: HarnessEntityStore, groundTruth: GroundTruth): GraphScore`
- **Entity precision**: (correctly detected entities) / (all detected entities). Match by canonicalName or alias overlap with ground truth entities.
- **Entity recall**: (correctly detected entities) / (ground truth entities). How many ground truth entities were found.
- **Relationship precision**: (correctly inferred relationships) / (all inferred relationships). Match by entity name + relationship type.
- **Relationship recall**: (correctly inferred relationships) / (ground truth relationships). How many ground truth relationships were found.
- **Privacy score**: For each ground truth entity with a known relationship, check if the store has an EntityRelation linking that entity. If yes, sanitization could use "[SPOUSE]" instead of "[PERSON]". Privacy score = (entities with inferred relationships) / (ground truth entities with relationships).
- Return `GraphScore`: `{ entityPrecision, entityRecall, entityF1, relationshipPrecision, relationshipRecall, relationshipF1, privacyScore, totalEntities, totalRelations, checkpoint }`

Create `scripts/harness/write-reports.ts` — generates JSON + Markdown reports:

- `writeReports(results: CheckpointResult[], outputDir: string)`:
- **JSON report**: Full checkpoint data array with timestamp, written to `scripts/harness/reports/harness_{timestamp}.json`
- **Markdown report**:
  - Summary table: checkpoint | entities P/R/F1 | relationships P/R/F1 | privacy score
  - Learning curve ASCII chart: show entity recall and relationship recall progression across checkpoints using simple bar chars (e.g., `|████████░░░░| 67%`)
  - Entity detail: which ground truth entities were found/missed at final checkpoint
  - Relationship detail: which relationships were correctly inferred, which missed
- Written to `scripts/harness/reports/harness_{timestamp}.md`
- Create `scripts/harness/reports/` directory if it doesn't exist
