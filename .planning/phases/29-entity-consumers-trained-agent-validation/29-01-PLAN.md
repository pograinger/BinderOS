**enrichment-emulator.ts** — Cloud-as-user enrichment Q&A simulation:
- `emulateEnrichmentSession(context: HarnessEnrichmentContext, personaBio: string, client: Anthropic): Promise<HarnessEnrichmentResult>`
- HarnessEnrichmentContext: atomContent, atomType, entitySummary (known entities + relationships as text), priorQA (existing enrichment records)
- Uses Haiku (`claude-haiku-4-5-20250514`) for cost efficiency — per CONTEXT.md locked decision
- Prompt: "You are {personaName}. Here is a GTD inbox item you captured. Answer 3 enrichment questions naturally as yourself. Do NOT directly state relationship types — answer as you would naturally." Include atom content, entity summary, and prior Q&A
- Returns: simulatedQA (3 question/answer pairs), plus runs `runHarnessKeywordPatterns()` and `updateHarnessCooccurrence()` on each answer text to mine entity signals — this is the enrichment answer mining
- Entity summary builder: `buildEntitySummary(store: HarnessEntityStore, mentionedEntityIds: string[]): string` — creates human-readable entity context block showing known entities and their relationships

**correction-emulator.ts** — Cloud-as-user correction simulation:
- `generateCorrections(store: HarnessEntityStore, groundTruth: GroundTruth, client: Anthropic): Promise<UserCorrection[]>`
- Uses Haiku to review current entity graph vs ground truth
- For each wrong relationship (type mismatch between store and GT), generates a correction
- `applyCorrection(store: HarnessEntityStore, correction: UserCorrection)`: updates entity relation to confidence 1.0, sourceAttribution='user-correction'. Then re-runs `runHarnessKeywordPatterns()` on ALL atoms that mention the corrected entity — corrections suppress conflicting inferred relations via `cleanSuppressedRelations()`
- Critical: user-corrections (sourceAttribution='user-correction') NEVER get overwritten by inference. Check this in harness-inference.ts — if a relation already has sourceAttribution='user-correction', skip pattern upsert for that entity+type

**adversarial-cycle.ts** — Single adversarial cycle orchestrator:
- `runAdversarialCycle(persona, cycleNumber, previousGaps, store, client): Promise<CycleState>`
- Cycle 1: uses existing generate-corpus.ts pattern to create natural corpus (30-50 items)
- Cycles 2-5: gap-targeted generation via Sonnet — prompt includes persona profile, current graph state, exact missed relationships (previousGaps), instructs: "Generate 30-40 adversarial items targeting THESE missing relationships. Use varied phrasing, indirect references, cultural naming variations, implicit relationship evidence. Do NOT use obvious keywords — test the limits of pattern matching."
- For each corpus item: (1) `runHarnessAtom(item, store)`, (2) `emulateEnrichmentSession()` with entity context, (3) mine answers for entities
- After all items: `flushHarnessCooccurrence(store)` + `cleanSuppressedRelations(store)`
- Then: `generateCorrections()` + `applyCorrection()` for each
- Score graph, take snapshot, compute diff vs previous cycle, extract gaps
- Save checkpoint via checkpoint-store
- Component attribution tracking: for each new relation created during the cycle, record whether it came from keyword-patterns, co-occurrence, enrichment-mining, or user-correction. Store in CycleState.attribution
- Synthetic timestamps: cycle 1 items get timestamps from ~4 weeks ago, cycle 5 from ~now. Linear interpolation across cycles. Pass to HarnessEntityStore entity.lastSeen updates

**harness-pipeline.ts modifications:**
- Add optional `syntheticTimestamp` parameter to `runHarnessAtom()` — when provided, use it for entity.lastSeen and atomIntelligence timestamps instead of Date.now()
- Export `getProcessedAtomIds(store)` helper for correction ripple (re-evaluate existing atoms)

**harness-inference.ts modifications:**

- Export `reRunPatternsForEntity(entityId, store)`: finds all atoms mentioning this entity in store's atomIntelligence, re-runs keyword patterns on each — used by correction ripple

- For each persona: create fresh HarnessEntityStore, resetHarnessCooccurrence(), run 5 adversarial cycles, collect PersonaAdversarialResult
- Resume logic: check experiment state file, skip completed personas, resume mid-persona via cycle checkpoints
