- `computeEntityRelevance(mentionCount: number, lastSeenMs: number, nowMs?: number): number`
- Formula: `mentionCount * Math.exp(-(Math.LN2 / 30) * daysSinceLastSeen)`
- Export `HALF_LIFE_DAYS = 30` constant

**src/entity/recency-decay.test.ts** — TDD tests:
- Test zero days = no decay
- Test 30 days = ~50% decay (half-life)
- Test 60 days = ~25%
- Test 365 days = near zero
- Test mentionCount scaling (higher count = higher relevance at same age)

**src/storage/entity-helpers.ts** — Add three new exported functions:
- `correctRelationship(entityId: string, correctType: string, atomId: string): Promise<void>` — finds existing relations for entity, removes any that conflict with the correction, creates new relation with `sourceAttribution: 'user-correction'`, `confidence: 1.0`, `evidence: ['user-correction from atom ' + atomId]`. Uses `[SELF]` as sourceEntityId per Phase 28 convention
- `getEntityTimeline(entityId: string): Promise<string[]>` — queries `db.atomIntelligence` where entityMentions contains this entityId. Returns atomIds sorted by atom createdAt descending. Uses Dexie `.filter()` on entityMentions array since there's no direct index on nested entity IDs
- `findHighestConfidenceRelation(entityText: string): Promise<EntityRelation | null>` — finds entity by canonicalName or alias match, then finds all relations where entity is source or target. Returns user-corrections first (confidence 1.0), then highest confidence >= 0.6 among inferred. Returns null if no qualifying relation
