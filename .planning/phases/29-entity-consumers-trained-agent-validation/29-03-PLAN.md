**ablation-engine.ts** — Component ablation testing:
- `runAblation(persona: PersonaPaths, config: AblationConfig, existingCorpora: CorpusItem[][], client: Anthropic): Promise<AblationResult>`
- Reuses pre-generated corpora from the full run (per research: ablation doesn't regenerate corpus, just re-runs pipeline with disabled component). This makes ablation cheap — no new API calls for corpus generation
- Creates fresh HarnessEntityStore, runs through all cycle corpora sequentially with the ablation config applied
- AblationResult: config (which component disabled), finalScore (GraphScore), perCycleScores (GraphScore[]), comparisonToFull (delta vs full-run final score)

- `runFullAblationSuite(personas: PersonaAdversarialResult[], client: Anthropic): Promise<AblationSuiteResult>`
- Per research cost control: run ablation only on 2 representative personas (one low-complexity, one high-complexity) using 3 cycles instead of 5
- Tests 5 ablation configs (one per component disabled): disableKeywordPatterns, disableCooccurrence, disableEnrichmentMining, disableUserCorrections, disableRecencyDecay
- AblationSuiteResult: fullRunScores, perComponentResults (Map<componentName, AblationResult[]>), componentRanking (sorted by impact: delta between full run and ablated run)

**adversarial-cycle.ts modification:**
- Add optional `ablationConfig?: Partial<AblationConfig>` parameter to `runAdversarialCycle()`
- When `ablationConfig.disableKeywordPatterns = true`: skip `runHarnessKeywordPatterns()` calls
- When `ablationConfig.disableCooccurrence = true`: skip `updateHarnessCooccurrence()` and `flushHarnessCooccurrence()` calls
- When `ablationConfig.disableEnrichmentMining = true`: skip entity mining on enrichment answers (still generate Q&A for realism, just don't mine)
- When `ablationConfig.disableUserCorrections = true`: skip `generateCorrections()` and `applyCorrection()` calls
- When `ablationConfig.disableRecencyDecay = true`: flag in scoring (no direct inference effect)

**auto-tune-patterns.ts** — Cross-persona pattern optimization:
- `autoTunePatterns(results: PersonaAdversarialResult[]): TuneResult`
- Reads `src/config/relationship-patterns.json` (original patterns)
- For each pattern, computes cross-persona precision: (times pattern fired AND produced correct relationship) / (times pattern fired total). Uses CycleState.attribution to track which patterns fired
- Patterns with precision > 70%: increase `confidenceBase` by 0.05 (capped at 0.95)
- Patterns with precision < 40%: halve `confidenceBase` and add to `flags[]` array with reason
- Pattern suggestion: collect all `missedRelations` across all personas and cycles. For each missed relationship, find corpus items that SHOULD have evidenced it. Use Sonnet to analyze those items and suggest new keyword patterns. Format as { keyword, relationshipType, suggestedConfidence, evidence }
- Output: writes `scripts/harness/tuned-patterns.json` — same schema as relationship-patterns.json but with adjusted values + `_tuning` metadata block (timestamp, personas used, precision stats per pattern)
- Also writes `scripts/harness/pattern-suggestions.json` — new patterns suggested by analysis

**generate-investment-report.ts** — Actionable investment recommendations:
- `generateInvestmentReport(experiment: ExperimentResult, ablation: AblationSuiteResult, tuneResult: TuneResult, client: Anthropic): Promise<string>`
- Uses Sonnet to synthesize findings into a structured investment report
- Report sections:
  1. Executive Summary: overall stack performance, pass/fail against 80% threshold, learning curve shape (logarithmic, linear, or stalled)
  2. Component Attribution: which components carry what % of relationships, ablation-derived importance ranking
  3. Gap Analysis: common missed relationship categories across all personas, per-archetype weaknesses
  4. Recommendations: ranked list of InvestmentItem objects with title, description, expectedAccuracyGain, implementationComplexity (LOW/MED/HIGH), dependencies, derivedFrom
  5. Pattern Tuning Summary: which patterns improved, which flagged, suggested new patterns
  6. Cross-Persona Consistency: standard deviation of relation F1, identify personas that significantly underperform and why
  7. Enrichment Quality: entity context injection value measured by enrichment quality comparison scores
- InvestmentItem examples derived from ablation: "If keyword patterns carry 60% of relationships but gap is temporal reasoning, recommend temporal ONNX agent with expected +15% gain"
- Output: `scripts/harness/experiments/{name}/investment-report.md` as formatted Markdown

**run-adversarial.ts** — Add post-run analysis phase:
- After all personas complete their 5 cycles: (1) run full ablation suite, (2) auto-tune patterns, (3) generate investment report
- New flags: `--skip-ablation` (skip ablation for faster runs), `--skip-report` (skip investment report)
- Print summary table to stdout: per-persona final F1, aggregate metrics, pass/fail status, ablation highlights
- Save complete experiment: `scripts/harness/experiments/{name}/` with summary.json, investment-report.md, ablation-results.json, tuned-patterns.json

**write-reports.ts** — Final experiment report with ablation:

- Add learning curve classification: if F1 increases monotonically = "healthy logarithmic"; if plateaus before cycle 5 = "early saturation"; if decreases = "degradation" (red flag)
