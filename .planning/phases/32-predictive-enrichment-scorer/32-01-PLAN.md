    Create `src/ai/enrichment/momentum-builder.ts`:
    - Import db from `../../storage/db`, CachedCognitiveSignal from types, MomentumVector from predictive-scorer
    - Module-level `_predictionCache = new Map<string, { result: MomentumVector; entityScores: Record<string, number>; timestamp: number }>()`
    - Module-level `_invalidationLog: Array<{ binderId: string; reason: string; timestamp: number }> = []`

    **computeMomentumVector(binderId, config)** — async function:
    - Check cache: if entry exists and `Date.now() - entry.timestamp < config.cacheTtlMs`, return cached `{ momentum: entry.result, entityScores: entry.entityScores }`
    - Fetch atom IDs for binder: `db.atoms.where('binderId').equals(binderId).primaryKeys()` (returns string[])
    - Batch-get atomIntelligence rows: `db.atomIntelligence.where('atomId').anyOf(atomIds).toArray()`
    - Filter to rows with `cognitiveSignals.length > 0` (Pitfall 3 — empty arrays don't count)
    - Apply hybrid window: filter by `lastUpdated >= cutoffMs` (where cutoffMs = Date.now() - config.maxWindowHours * 3600000), then sort by lastUpdated descending, take first `config.windowSize`
    - Count atoms with signals for cold-start: `coldStart = filteredRows.length < config.coldStartThreshold`
    - Build momentum vector using exponential decay adapted from `computeEntityRelevance()`:
      - For each atom at position `i` from most recent (0 = newest):
        - `weight = Math.exp(-(Math.LN2 / config.momentumHalfLife) * i)`
        - For each CachedCognitiveSignal in atom's cognitiveSignals:
          - Key: `signal.modelId` (use modelId directly, not modelId_label, since label is the topLabel for that model)
          - `signalFrequency[key] += weight` (label appeared)
          - `signalStrength[key] += weight * signal.confidence`
    - Build MomentumVector: `{ signalFrequency, signalStrength, entityScores: {}, coldStart, atomCount: filteredRows.length }`

    **computeEntityTrajectory(binderId, atomEntityIds, config)** — async function:
    - `atomEntityIds`: entity IDs detected in the current atom (pre-filtered by caller)
    - If `atomEntityIds.length === 0`, return empty `Record<string, number>`
    - Count atoms with entity mentions for entity cold-start: query atomIntelligence rows with `entityMentions.length > 0`; if count < `config.entityColdStartThreshold`, return empty
    - For each entity in atomEntityIds:
      - Query Entity table for mentionCount and lastSeen
      - Compute trajectory using adapted recency decay: `mentionCount * exp(-(LN2 / config.momentumHalfLife) * daysSince)`
      - Check EntityRelation table for user-correction: if any relation with sourceAttribution === 'user-correction' involves this entity, multiply score by 2.0
      - Multiply by entity type priority weight from config (`config.entityTypePriorityWeights[entity.type] ?? 1.0`)
    - Return `Record<entityId, number>` scores

    **Cache management exports:**
    - `invalidateCache(binderId: string, reason?: string)`: delete from _predictionCache, push to _invalidationLog
    - `getCacheState(binderId: string)`: return cache entry or undefined
    - `getInvalidationLog()`: return _invalidationLog (for harness)
    - `clearInvalidationLog()`: reset _invalidationLog (for harness)

    Cache the combined result (momentum + entityScores) on successful computation.
