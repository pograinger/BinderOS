  <name>Task 1: Ring buffer + LSTM session in embedding worker</name>
  <files>src/search/embedding-worker.ts, src/search/__tests__/ring-buffer.test.ts</files>
  <behavior>
    - updateRingBuffer('binder-1', [0.1, 0.2, ...384], 5) stores embedding; subsequent calls push to buffer; 6th call evicts oldest
    - getRingBuffer('binder-1') returns current buffer contents as number[][]
    - LOAD_RING_BUFFER message populates in-memory buffer from provided embeddings array
    - GET_SEQUENCE_CONTEXT with empty buffer returns 128-dim zero vector
    - GET_SEQUENCE_CONTEXT with loaded buffer and loaded LSTM session returns 128-dim non-zero vector
    - RING_BUFFER_UPDATED message is posted back to main thread after each buffer update with full buffer state
    - Sequence model not found gracefully returns zero-padded context (no error thrown)
  </behavior>
  <action>
    1. Add in-memory ring buffer Map at module scope in embedding-worker.ts:
       ```typescript
       const ringBuffers = new Map<string, number[][]>();
       ```

    2. Add ring buffer helper functions (updateRingBuffer, getRingBuffer) — pure functions operating on the Map.

    3. Add SEQUENCE_MODEL ClassifierConfig entry:
       ```typescript
       const SEQUENCE_MODEL: ClassifierConfig = {
         name: 'sequence-context',
         modelPath: 'models/sequence-context.onnx',
         classesPath: '',
         session: null, classMap: null, loading: false,
       };
       ```

    4. Add `loadSequenceModel()` function following the lazy-load pattern (like loadCompletenessGate). Attempts to create ORT session from models/sequence-context.onnx. Sets `SEQUENCE_MODEL.session` on success. Logs warning and returns silently on failure (model may not exist yet).

    5. Add `runSequenceInference(embeddings: number[][]): Promise<number[]>` function:
       - If SEQUENCE_MODEL.session is null, return `new Array(128).fill(0)`
       - Otherwise: flatten embeddings to Float32Array of shape [seq_len, 1, 384], create ORT Tensor, run inference, return Array.from(results['context'].data)

    6. Add message handlers in the `self.onmessage` handler:
       - `LOAD_RING_BUFFER`: `{ type: 'LOAD_RING_BUFFER'; binderId: string; embeddings: number[][] }` — sets ringBuffers.set(binderId, embeddings). No response needed.
       - `UPDATE_RING_BUFFER`: `{ type: 'UPDATE_RING_BUFFER'; binderId: string; embedding: number[]; windowSize: number }` — calls updateRingBuffer, then posts `{ type: 'RING_BUFFER_UPDATED'; binderId: string; embeddings: number[][] }` back to main thread.
       - `GET_SEQUENCE_CONTEXT`: `{ type: 'GET_SEQUENCE_CONTEXT'; id: string; binderId: string; windowSize: number }` — reads ring buffer for binderId, calls runSequenceInference, posts `{ type: 'SEQUENCE_CONTEXT_RESULT'; id: string; context: number[] }`. On error posts `{ type: 'SEQUENCE_CONTEXT_ERROR'; id: string; error: string }`.

    7. Update the JSDoc message protocol comment at top of file to document the 3 new incoming and 3 new outgoing message types.

    8. Write ring-buffer.test.ts testing the pure ring buffer logic (extracted functions). Test: add to buffer, cap at N, cold-start returns empty, LOAD_RING_BUFFER populates buffer. Mock the worker postMessage for message handler tests.
  </action>

  </verify>
