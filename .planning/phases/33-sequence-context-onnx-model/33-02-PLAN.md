<task type="auto">
  <name>Task 1: Sequence data generation + LSTM training + ONNX export + validation</name>
  <files>scripts/train/sequence/60_generate_sequence_data.py, scripts/train/sequence/61_train_sequence_model.py, scripts/train/sequence/62_validate_sequence_model.mjs</files>
  <action>
    **Pre-requisite check:** Before running any script, verify torch is installed: `python -c "import torch; print(torch.__version__)"`. If not installed, run: `pip install torch --index-url https://download.pytorch.org/whl/cpu onnxruntime sentence-transformers`

    **Script 60_generate_sequence_data.py:**
    1. Load all persona corpus files from `scripts/harness/personas/*/corpus.json`
    2. For each persona, extract atom content strings in chronological order
    3. Embed each atom using sentence-transformers MiniLM (normalize_embeddings=True, .astype(np.float32))
    4. Build training pairs for self-supervised next-embedding prediction:
       - For each sequence of length >= 2: input = embeddings[0:i], target = embeddings[i] (for i in 1..len)
       - Pad/truncate input sequences to max window sizes tested (3, 5, 7)
       - Generate sequences for each window size: N=3, N=5, N=7
    5. Save to `scripts/train/sequence/sequence_training_data.npz` with arrays: X_train (sequences), y_train (next embeddings), window_sizes

    **Script 61_train_sequence_model.py:**
    1. Define SequenceContextModel (PyTorch nn.Module):
       - LSTM: input_size=384, hidden_size=64, num_layers=1, batch_first=False
       - Linear projection: 64 -> 128 (output_size)
       - Forward: x shape (seq_len, 1, 384) -> LSTM -> h_n (1, 1, 64) -> squeeze -> Linear -> (128,)
    2. Training objective: next-embedding prediction (MSE loss between projected hidden state and target embedding's first 128 dims, or a learned projection of the 384-dim target)
       - Actually simpler: train LSTM to predict next embedding projected to 128-dim. The 128-dim space IS the sequence context.
       - Alternative: train end-to-end with cosine similarity loss between projected output and target embedding. This teaches the LSTM to compress sequence information into a direction-preserving 128-dim vector.
    3. Train for 50-100 epochs with Adam optimizer, lr=1e-3, grad clipping at 1.0
    4. Export with torch.onnx.export:
       ```python
       torch.onnx.export(model, dummy, "public/models/sequence-context.onnx",
                         dynamo=True, fallback=True, opset_version=18,
                         input_names=["embeddings"], output_names=["context"],
                         dynamic_axes={"embeddings": {0: "seq_len"}})
       ```
    5. Validate immediately: onnx.checker.check_model(), onnxruntime inference with seq_len=3,5,7
    6. Print model file size and verify < 500KB (512,000 bytes)
    7. Save frozen LSTM for feature extraction by classifier retraining script

    **Script 62_validate_sequence_model.mjs:**
    1. Load sequence-context.onnx via onnxruntime-node
    2. Create test inputs at seq_len=1, 3, 5, 7 (all with batch=1, dim=384)
    3. Verify output shape is [128] for each input
    4. Verify outputs are not all zeros (model is producing real context)
    5. Print pass/fail for each test case
    6. Follow existing pattern from scripts/train/04_validate_model.mjs

    Use `python -u` for all Python script execution (unbuffered output per MEMORY.md).
  </action>

  </verify>
  <done>sequence-context.onnx exists in public/models/, is under 500KB, validates with onnxruntime-node for variable seq_len inputs, and produces non-zero 128-dim output.</done>

<task type="auto">
  <name>Task 2: Retrain all T2 classifiers with 512-dim input + validation</name>
  <files>scripts/train/sequence/63_retrain_classifiers_512.py, scripts/train/sequence/64_validate_classifiers_512.mjs</files>
  <action>
    **Script 63_retrain_classifiers_512.py:**
    1. Load the frozen LSTM model (from Task 1 output) to generate 128-dim context vectors
    2. For each existing classifier training dataset (from scripts/training-data/*.jsonl and scripts/train/cognitive_*_test_embeddings.json):
       a. Load the existing 384-dim embeddings (already pre-computed in prior training runs)
       b. For each embedding, generate a 128-dim sequence context:
          - 40-50% of samples: zero-padded 128-dim (cold-start augmentation)
          - Remaining: run through frozen LSTM with a random-length window (1 to 7) of embeddings from the same training set as pseudo-context
       c. Concatenate [384-dim, 128-dim] -> 512-dim
       d. Train MLPClassifier with same hyperparameters as existing scripts (hidden_layer_sizes, max_iter, etc.)
       e. Wrap in CalibratedClassifierCV(method='sigmoid')
       f. Export with skl2onnx: FloatTensorType([None, 512]), target_opset=17, zipmap=False
       g. Save to public/models/classifiers/ with same filenames as existing (overwriting — but keep originals backed up as *-384.onnx)
    3. Classifiers to retrain (all that use runClassifierOnEmbedding):
       - type-classifier (triage type)
       - gtd-routing, actionability, project-detection, context-tagging (GTD)
       - completeness-gate
       - decomposition
       - All 6 missing-info classifiers
       - All 10 cognitive models (if they use 384-dim input)
    4. Print F1 scores for each retrained classifier on held-out test set
    5. Back up original 384-dim ONNX files as *-384-backup.onnx before overwriting

    **Script 64_validate_classifiers_512.mjs:**
    1. Load each retrained 512-dim classifier via onnxruntime-node
    2. Create test input of shape [1, 512] (random float32)
    3. Verify output has probability distribution (values sum to ~1.0)
    4. Verify same class labels in classes.json still work
    5. Print pass/fail per classifier

    **Important:** This task depends on Task 1 (needs the frozen LSTM). Execute sequentially within this plan.
  </action>

  </verify>
