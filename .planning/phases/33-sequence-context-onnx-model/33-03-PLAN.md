<task type="auto">
  <name>Task 1: Ablation script — F1 comparison across window sizes</name>
  <files>scripts/train/sequence/65_ablation_sequence.py</files>
  <action>
    Create `scripts/train/sequence/65_ablation_sequence.py`:

    1. **Setup:** Load sentence-transformers MiniLM, load frozen LSTM from 61_train output, load all classifier test datasets (same test sets used in Plan 02 training — must be identical splits for valid comparison per Research pitfall 7).

    2. **Baseline measurement (384-dim):**
       - For each backed-up 384-dim classifier (*-384-backup.onnx), compute F1 on test set using onnxruntime Python
       - Store as baseline_f1[classifier_name]

    3. **Sequence context measurement (512-dim):**
       - For each window size N in [3, 5, 7]:
         a. For each test sample, build a synthetic sequence context:
            - Sample N-1 random embeddings from the training set as "prior context"
            - Prepend them to the test embedding to form a sequence of length N
            - Run through frozen LSTM to get 128-dim context
         b. Concatenate [384-dim test embedding, 128-dim context] -> 512-dim
         c. Run through 512-dim classifier ONNX model
         d. Compute F1 on test set
         e. Store as seq_f1[classifier_name][N]

    4. **Report generation:**
       - For each classifier and window size, compute delta = seq_f1 - baseline_f1
       - Flag classifiers where ALL window sizes show improvement (candidates for replacement)
       - Flag classifiers where ANY window size shows degradation (caution)
       - Identify best N per classifier
       - Compute aggregate: mean F1 delta across all classifiers per N
       - Determine recommended window size (N with best aggregate improvement)
       - Write `scripts/train/sequence/ablation_report.json`:
         ```json
         {
           "timestamp": "...",
           "window_sizes_tested": [3, 5, 7],
           "baseline_dim": 384,
           "sequence_dim": 512,
           "classifiers": {
             "type-classifier": {
               "baseline_f1": 0.92,
               "seq_f1": {"3": 0.93, "5": 0.94, "7": 0.93},
               "best_n": 5,
               "best_delta": 0.02,
               "recommend_replace": true
             }
           },
           "aggregate": {
             "mean_delta": {"3": 0.01, "5": 0.02, "7": 0.015},
             "recommended_n": 5,
             "overall_recommendation": "replace|keep_384"
           }
         }
         ```
       - Also write `scripts/train/sequence/ablation_report.md` with human-readable table

    5. **Decision output:** Print clear recommendation to stdout:
       - "RECOMMEND: Replace production classifiers with 512-dim models (N=5, mean F1 delta: +0.02)"
       - OR "KEEP: 384-dim classifiers — sequence context did not improve F1 (mean delta: -0.01)"

    Use identical train/test splits as 63_retrain_classifiers_512.py (same random seed, same data). This is critical per Research pitfall 7.
  </action>
  <verify>
    <automated>python -u scripts/train/sequence/65_ablation_sequence.py</automated>
  </verify>
