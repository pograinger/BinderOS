    1. Create `src/ai/feature-vectors/vector-cache.ts`:
       - `shouldRecomputeVector(atom: Atom, cached: CanonicalVector | undefined): boolean`
         - Returns true if cached is undefined.
         - Returns true if atom.updated_at > cached.lastComputed.
         - Otherwise false (cosmetic edit didn't change vector-feeding fields).

       - `dirtyCheckTaskFields(prev: Partial<TaskAtom>, next: TaskAtom): boolean`
         - Compares: dueDate, status, energy, context, links.length. Returns true if any differ.
         - Used by callers who have the previous atom state to avoid recomputation on cosmetic edits.

       - `writeCanonicalVector(atomId: string, vectorType: 'task' | 'person' | 'calendar', vector: Float32Array): void`
         - Fire-and-forget async wrapper (same pattern as writePredictionMomentum).
         - Converts Float32Array to number[] via `Array.from(vector)`.
         - Writes to atomIntelligence.canonicalVector: `{ vectorType, data: Array.from(vector), lastComputed: Date.now(), schemaVersion: 1 }`.
         - Catches errors, logs console.warn, never throws.

       - `recomputeAndCacheVector(atom: Atom, sidecar: AtomIntelligence | undefined, entities: Entity[], relations: EntityRelation[]): void`
         - Determines atom type → calls appropriate compute function.
         - For type 'task': `computeTaskVector(atom as TaskAtom, sidecar, entities, relations)` → `writeCanonicalVector(atom.id, 'task', result)`.
         - For type 'event': `computeCalendarVector(atom as EventAtom, sidecar, entities, relations)` → `writeCanonicalVector(atom.id, 'calendar', result)`.
         - For other atom types: no-op (person vectors are computed from Entity, not Atom — handled separately).
         - Synchronous compute + async fire-and-forget write.

       - `recomputePersonVector(entity: Entity, relations: EntityRelation[]): void`
         - Calls computePersonVector → writeCanonicalVector with a synthetic atomId derived from entity.id (e.g., `entity:${entity.id}`).
         - Person atoms may not have a 1:1 atomId mapping — use entity.id as the key.
