Create `src/ai/eii/index.ts` with:
- `computeEII(results: ConsensusResult[], impact?: number): EIIResult` — pure function, no Dexie imports.
  - If results.length === 0, return zeroes.
  - Coherence = `Math.sqrt(variance)` of `results.map(r => r.weightedProbability)` — std-dev, NOT AUC (per user decision).
  - Stability = `results.reduce((sum, r) => sum + r.agreementScore, 0) / results.length` — mean agreement.
  - Impact defaults to 0 (production has no labels; harness caller passes computed impact).
  - EII = `(coherence + stability + impact) / 3` — equal weights per user decision.
- `updateBinderEII(binderId: string): Promise<void>` — queries `db.atomIntelligence` for all rows with `consensusRisk` for this binder (filter by non-null `consensusRisk`), maps to ConsensusResult[], calls `computeEII()`, writes to `db.binderIntelligence.put()`.
  - Import `db` from `../../storage/db`.
  - Import `ConsensusResult` from `../consensus/types`.
  - Full recompute strategy (not incremental) — simpler, correct, and the atomIntelligence table for a single binder is small enough that scanning it is fast.

Create `src/storage/migrations/v11.ts` following v10 pattern exactly:
- `export function applyV11Migration(db: BinderDB): void`

- Call `applyV11Migration(this)` after `applyV10Migration(this)` in the constructor.

Note: The `BinderIntelligenceEntry` in the v11 migration matches `BinderEIISnapshot` — use the same type. The table stores only the latest snapshot per binder (one row per binder, overwritten on each update, per user decision).
