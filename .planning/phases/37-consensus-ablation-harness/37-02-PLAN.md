**harness-consensus.ts — production consensus wrapper for harness context:**

Create `scripts/harness/harness-consensus.ts`:

1. Import `computeConsensus` from `../../src/ai/consensus/consensus-voter.js` (pure, no DOM).
   Import `SPECIALIST_WEIGHTS, SPECIALIST_FEATURE_SLICES` from `../../src/ai/consensus/types.js`.
   Import `computeEII` from `../../src/ai/eii/index.js`.
   Import `HarnessONNXSessions, runSpecialistInference` from `./harness-onnx.js`.
   Import types: `ConsensusResult, SpecialistOutput` from consensus types, `EIIResult` from eii types.

2. Export `async function runHarnessConsensus(sessions: HarnessONNXSessions, fullVector: number[]): Promise<ConsensusResult>`:
   - For each specialist in SPECIALIST_FEATURE_SLICES:
     - Call `runSpecialistInference(sessions, fullVector, slice.name, slice.featureIndices)` to get probability.
     - Build `SpecialistOutput { name, probability, weight: SPECIALIST_WEIGHTS[name] }`.
   - Call `computeConsensus(outputs)` on the collected specialist outputs.
   - Return the ConsensusResult.

   CRITICAL: Do NOT import from `specialist-runner.ts` (it uses `new Worker(...)` which fails in Node.js). Call `computeConsensus()` from `consensus-voter.ts` directly.

3. Export `function deriveRiskLabels(corpus: CorpusItem[]): boolean[]`:
   - Map each corpus item to a risk boolean based on metadata.
   - An item is "risky" if ANY of:
     - Has a deadline AND deadline is within 7 days (check `metadata.deadline` or `metadata.dueDate`).
     - Has `metadata.waitingFor` or `metadata.status === 'waiting'`.
     - Has `metadata.priority === 'high'` or `metadata.priorityTier === 'critical'`.
   - Import CorpusItem type from `./generate-corpus.js`.
   - Document the threshold definition clearly.

4. Export `function computeHarnessImpact(riskLabels: boolean[], consensusProbs: number[], topKPct?: number): number`:
   - Default topKPct = 0.3.
   - Compute recall@k: fraction of truly risky atoms in the top-k consensus-predicted atoms.
   - Compute baseline recall: total risky / total atoms.
   - Impact = `(modelRecall - baselineRecall) / (1 - baselineRecall)`, clamped to [0, 1].
   - If no risky atoms exist, return 0.
   - This matches the formula from `eii-experiment.py` lines 527-546.

**ablation-engine.ts — add specialist ablation:**

Add new types at the top of the file:
```typescript
export interface SpecialistAblationResult {
  specialistRemoved: string;
  fullConsensusEII: number;
  ablatedConsensusEII: number;
  eiiDelta: number;           // ablated - full (negative = specialist was helping)
  accuracyDelta: number;      // true positive rate delta
  consensusLift: number;      // full consensus vs single specialist
}
```

Add export function `runSpecialistAblation(allConsensusResults: ConsensusResult[], riskLabels: boolean[]): SpecialistAblationResult[]`:
- This is entirely POST-HOC — zero re-inference cost. It filters stored `specialistContributions[]` and re-calls `computeConsensus()`.
- Import `computeConsensus` from `../../src/ai/consensus/consensus-voter.js`.
- Import `computeEII` from `../../src/ai/eii/index.js`.
- Import `computeHarnessImpact` from `./harness-consensus.js`.
- Get specialist names from the first ConsensusResult's specialistContributions.
- Compute full EII: `computeEII(allConsensusResults, computeHarnessImpact(riskLabels, allConsensusResults.map(r => r.weightedProbability)))`.
- For each specialist name, leave-one-out:
  - Filter each ConsensusResult's specialistContributions to exclude that specialist.
  - Re-call `computeConsensus()` on remaining outputs.
  - Guard: if remaining.length === 0, skip (keep original).
  - Compute ablated EII with ablated results + ablated impact.
  - `eiiDelta = ablatedEII - fullEII` (negative = specialist was helping, matching existing convention from `computeAblationDelta`).
  - `consensusLift = fullEII - singleSpecialistEII` (run computeEII on results with only that one specialist — proves ensemble > single).
- Return sorted by `Math.abs(eiiDelta)` descending (most impactful first).
